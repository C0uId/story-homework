## 1. Demo 简介

**是什么：** 一个 AI 驱动的故事化学习网站（Vue 3 单页应用），把知识点编成武侠、侦探、科幻等冒险风格的分集短剧，学生在"追剧"过程中答题闯关，不知不觉掌握知识。

**面向谁：** 小学 1-6 年级学生，以及想让孩子主动学习的家长、想给课堂增加趣味性的老师。

**核心功能：**

| 功能             | 说明                                                |
| -------------- | ------------------------------------------------- |
| 🎭 **5 种冒险风格** | 武林秘籍（武侠）、怪盗对决（怪盗基德风）、名探事件簿（柯南风）、星际探险（科幻）、校园奇遇（日常） |
| 📚 **全学科覆盖**   | 数学、语文、英语、科学、历史，6 个年级共 1920 集互动短剧                  |
| 🎯 **双模式学习**   | 片段式（单知识点体验）+ 持久式（整章长篇连载，像追剧一样）                    |
| ✍️ **答题闯关**    | 每集嵌入互动题目，答对推动剧情，答错有解析提示                           |
| 🌈 **暖色卡通 UI** | 圆角卡片、浮动 emoji 装饰、弹性动画，视觉上降低学习压力感                  |

## 2. Demo 创作思路

**灵感来源：**

身边很多中小学生对学习提不起劲，但刷短视频却能刷几个小时。核心区别在于：短视频有故事、有悬念、有情绪起伏，而作业什么都没有。我在想——如果把知识点编成一个有剧情的故事，让学生在"追剧"的过程中不知不觉把知识学了，会不会有效？

**想解决的问题：**

中小学生每天面对千篇一律的练习题和试卷，知识点被拆成孤立的碎片，学生不知道"为什么学"、"学了有什么用"。长期下来，学习变成了机械重复，热情被消磨殆尽。作业不是太少，而是太无聊。

**为什么做这个方向：**

我观察到三个关键事实：

1. 故事是最好的记忆载体——人对叙事性内容的记忆远强于抽象知识点
2. 游戏化是吸引注意力的有效手段，但很多教育产品只做表面（积分/勋章），忽略了内容本身的吸引力
3. AI 的出现让"个性化趣味内容"的生成成本骤降，以前需要专业编剧做的事情，现在一个人用 TRAE 就能完成

**我的核心取舍：** 不做一个"看起来好玩"的学习工具，而是做一个"真正好看"的内容产品——让内容本身像网剧一样让人想追下去，学习是副产品实现潜移默化中学习。

## 3. 用 TRAE 开发的完整过程

### 第一阶段：从想法到可运行的原型（约 2 小时）

最初只有一个模糊的想法——"用故事包装知识点"。我在 TRAE 中描述了这个想法，Trae AI 助手快速生成了一个 Vue 3 + Vite 的基础脚手架，包含：

- 首页课程选择界面
- 基础的故事播放页面（打字机效果 + 答题交互）
- 一份示例故事数据（单个章节）

这个原型让我立刻能"摸到"产品的样子，而不是停留在脑补阶段。看到实际效果后，我马上发现：**单种风格太单调了，学生可能看一章就腻了。**

### 第二阶段：快速迭代扩展功能（约 3 小时）

基于第一版原型，我通过自然语言向 TRAE 提出需求：

1. **"增加多种冒险风格"** → 加入了武侠、怪盗、侦探、科幻、校园 5 种风格，每种风格有不同的 UI 主题色和故事框架
2. **"对应年级要有对应内容"** → 构建了知识图谱映射（knowledgeMap.js），6 个年级 × 5 学科 × 4 章节 × 知识点
3. **"完善功能，不只是片段"** → 实现了持久式模式（整章连贯故事）和片段式模式（单知识点体验）
4. **"代码分离到独立项目"** → 从原始项目中将所有相关代码提取到独立的 `story-homework` 项目

每个需求只需要用中文描述，TRAE 就能理解意图并生成对应的代码。遇到错误时，把控制台的报错信息复制给 TRAE，它也能快速定位修复。

### 第三阶段：设计改版——暖色风格重设计（约 1.5 小时）

初始版本是暗色主题，虽然看起来很"酷"，但不太适合儿童教育场景。我找到一份参考风格（暖色调、圆角卡片、浮动装饰），用 TRAE 的 `design-taste-frontend` 技能，一次性重构了全部 UI：

- 背景色改为 `#FFF8F0` 暖白
- 卡片改用彩色边框（红/紫/蓝/青/橙对应 5 种风格）
- 加入浮动 emoji 装饰动画、弹入/弹跳等过渡效果
- 修复了布局居中问题

这次改版让我深刻体会到：**TRAE 不仅能写功能代码，还能做设计实现**——我只需要提供参考和描述，它就能把 CSS 和动画细节全部搞定。

### 第四阶段：内容填充——1920 集故事数据（约 4 小时）

这是最耗时的部分。我用 Python 脚本 + TRAE 辅助生成了大规模故事数据：

- 5 种风格 × 6 个年级 × 5 学科 × 4 章节 × 4 集/章节 × 多个故事段落 = **1920 集互动短剧**
- 每集包含：故事叙事文本、知识点标注、互动选择题（共 384 道精选题目）
- 故事数据约 239KB，全部在前端动态加载

TRAE 在这阶段帮助最大的是**数据处理脚本的编写和调试**——循环嵌套、正则匹配、JSON 格式校验等，反复迭代直到数据完全正确。

## 4. 技术架构

```
story-homework/
├── src/
│   ├── data/
│   │   ├── allStories.js       # 1920 集故事数据（239KB）
│   │   └── knowledgeMap.js     # 年级/学科/章节/知识点映射
│   ├── views/
│   │   ├── HomePage.vue        # 四步选择器（年级→学科→风格→章节）
│   │   ├── StoryPlayPage.vue   # 故事播放 + 答题交互
│   │   └── KnowledgeMapPage.vue# 知识地图（SVG 进度环）
│   ├── router/
│   │   └── index.js            # 路由配置
│   ├── style.css               # 全局暖色风格
│   └── App.vue
├── index.html                  # 入口
└── package.json                # Vite + Vue 3
```

**核心交互流程：**

```
用户选择年级 → 选择学科 → 选择冒险风格 → 选择章节/知识点
                                                     ↓
              答题闯关 ← 剧情推进 ← 打字机效果展示故事 ← 开始播放
               ↓
         答对 → 继续剧情
         答错 → 显示解析 → 重新作答
               ↓
           章节完成 → 解锁下一章（持久式）/ 回退选择（片段式）
```

**为什么选这个技术栈：**

- **Vue 3 + Vite**：轻量、热更新快、适合快速迭代原型
- **纯前端方案**：无需后端服务即可部署（Vercel/Netlify），降低参赛作品的部署门槛
- **动态 import 加载数据**：1920 集故事数据按需加载，首页秒开

## 5. 开发心得与经验总结

### 使用 TRAE 的感受

1. **"用自然语言编程"是真的**——整个项目我没有手写过一行 Vue 模板代码，全部通过描述需求让 TRAE 生成。我做得更多的是：提需求、看效果、反馈修改意见。
2. **TRAE 最擅长的是"搭架子"和"改样式"**——从零到可运行原型、从暗色到暖色改版，TRAE 都能快速完成。但在**复杂逻辑调试**（比如故事数据结构的嵌套匹配）和**大规模数据处理**（1920 集数据生成）上，仍然需要人工介入或配合脚本。
3. **报错信息是最好的提示词**——遇到 bug 时，直接把浏览器的 console 报错复制给 TRAE，它通常能快速定位到问题代码并给出修复方案，比自己一行行 debug 高效得多。

### 给参赛者的建议

- **先跑起来再说**：第一版不用完美，能运行最重要。有了实物再迭代优化。
- **用好截图**：Demo 帖中多放产品截图/动图，评审很难看完所有 Demo，亮眼的截图是加分项。
- **讲清楚"为什么"**：技术实现写得再详细，不如讲清楚你的洞察和取舍。评审更看重产品思维。
- **让 TRAE 做它擅长的**：UI 布局、样式调整、CRUD 代码生成——这些 TRAE 做得又快又好；算法逻辑、数据结构设计——这些还是需要你主导方向。
