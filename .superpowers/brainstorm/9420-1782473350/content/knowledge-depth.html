知识嵌入深度：知识在故事中的角色？

调研核心结论：知识必须是"通关的唯一手段"，而不是"故事中的附加题"。

3 种知识嵌入模型对比
A. 纯关卡模型（当前系统改良版）
结构：每幕 = 场景描述(短) → 知识关卡(答题) → 结果(继续/分支)
知识角色：知识 = 门锁密码 / 武器弹药 / 机关解法
叙事占比：30% 叙事 + 70% 互动
优点：学习密度高，每幕都有知识练习
缺点：节奏紧凑但缺少"呼吸感"，孩子可能觉得一直在做题
B. 故事优先模型（推荐）
结构：交替排列 — 纯故事幕 → 知识关卡幕 → 纯故事幕 → 知识关卡幕
知识角色：知识 = 推动剧情的"钥匙"，出现在关键转折点
叙事占比：50% 叙事 + 50% 互动
优点：有"追剧感"，孩子为了看剧情继续而主动答题
缺点：单集知识密度略低，但长期留存更好
C. 分支叙事模型
结构：每幕都有 2-3 个选择分支，选择决定剧情走向和知识点
知识角色：不同选择路径对应不同知识点，答错走"纠正支线"
叙事占比：60% 叙事 + 40% 互动
优点：沉浸感最强，选择真正有后果
缺点：AI 生成复杂度最高，分支一致性难保证
A

A. 纯关卡模型

高密度练习，每幕必答题。适合想快速提分的场景。

B

B. 故事优先模型（推荐）

故事和关卡交替，有"追剧感"。知识出现在关键转折点。

C

C. 分支叙事模型

选择驱动剧情，不同路径学不同知识。沉浸感最强但复杂度最高。
