知识剧场 - 重设计方案方向

基于调研结果，我识别出现有系统的 3 个核心问题和 3 个重构方向。请选择你最认可的方向。

现有系统的 3 个核心问题

问题诊断
1. 叙事太"教科书"
现有故事 narrative 要求 50-100 字且大量使用"你能帮帮我吗？"等固定句式。
→ 缺乏幽默感、情感张力和角色个性
2. 知识点是"附加题"而非"故事引擎"
问答和故事是分离的：先讲故事 → 然后出题。
→ 应该是"知识是通关的唯一手段"
3. 选择路径不够"有后果"
Wrong Path 是补充说明，不是真正的分支叙事。
→ 应该让选择真正改变故事走向

选择重构方向

A

🎯 方向一：故事引擎重构（推荐）

核心：重写 AI Prompt 系统 + 故事结构设计

  • 重写 system prompt → 用"知识即武器"原则
  • 新增 3 种叙事风格（幽默吐槽 / 悬疑解密 / 成长热血）
  • 重设 episode 结构：冲突 → 知识关卡 → 解决 → 新冲突
  • 分支叙事真正由选择驱动
  • 不改 UI，只改内容生成层
工作量：中等 · 效果：高 · 不破坏现有 UI
B

🎨 方向二：交互体验重构

核心：重做 UI 交互 + 增加游戏化机制

  • 首页从 4 步选择 → 一键"开始冒险"（自动匹配）
  • 新增"冒险地图"替代线性章节列表
  • 增加倒计时挑战、连击奖励等游戏化元素
  • 答对/答错的动画和音效反馈
  • 不改 AI 生成，只改前端交互
工作量：高 · 效果：中高 · 大量 UI 改动
C

🚀 方向三：全面重构（故事 + 交互 + 系统）

核心：整合方向 A + B，同时新增学习分析系统

  • 包含方向 A 的所有故事引擎改进
  • 包含方向 B 的部分交互改进
  • 新增"学习画像"：根据答题数据自动调整难度
  • 新增"班级排行"等社交功能
  • 最大改动量，可能需要 3-4 轮迭代
工作量：最高 · 效果：最高 · 建议分阶段实施
